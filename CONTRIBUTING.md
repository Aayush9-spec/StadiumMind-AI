# Contributing to StadiumMind AI

Thank you for your interest in contributing to **StadiumMind AI**! We welcome contributions from developers, designers, operations experts, and open-source enthusiasts.

## How to Contribute

### 1. Reporting Bugs
*   Check if the issue is already reported in the Issue Tracker.
*   If not, open a new issue detailing:
    *   Steps to reproduce the problem.
    *   Expected behavior vs. actual behavior.
    *   Screenshots or log output if applicable.

### 2. Suggesting Enhancements
*   Open an issue and detail the enhancement idea, target audience, and how it aligns with stadium operations.

### 3. Submitting Code Changes
1.  Fork the repository and create your branch from `main`.
2.  Install dependencies for both frontend and backend.
3.  Write clean, documented, and modular code.
4.  Add appropriate unit tests under `tests/`.
5.  Ensure accessibility (ARIA labels, keyboard navigation) guidelines are followed.
6.  Ensure no secrets, API keys, or private keys are committed.
7.  Run tests locally using `pytest` (backend) and build validation (frontend).
8.  Open a Pull Request with a clear description of the changes.

## Code Style & Standards

*   **Backend (Python/FastAPI):** PEP 8, Pydantic validation, clear docstrings, typing hints.
*   **Frontend (Next.js/React):** Tailwind CSS formatting, clean JSX structure, Framer Motion for premium animations, accessibility labels.
